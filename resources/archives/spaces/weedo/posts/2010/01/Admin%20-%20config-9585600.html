Admin - config

January 7 2010, 8:01 AM  by Ballut Vincent
Nous avons donc notre admin à l'adresse backend.php/questions. Lorsque ce module a été créé, un fichier generator.yml a également été généré, rangé dans le dossier config du module. C'est dans ce fichier que beaucoup de choses se passent.

Voilà le fichier généré :

https://gist.github.com/271262

Les vues

L'admin se compose à la base de trois "views" :
- list : le listing de tous les enregistrement
- new : formulaire d'ajout d'un nouvel enregistrement
- edit: formulaire de modification d'un enregistrement

Modification des champs

On va pouvoir modifié l'apparence et l'affichage de chaque champ, soit dans les trois views, soit dans chacune. Tout va se passer sous la ligne config.

Pour modifier un champ dans toutes les views, c'est la ligne "fields" qui entre en jeu :

https://gist.github.com/271267

Pour modifier le champ uniquement dans la view new (donc dans le formulaire d'ajout), il faut se placer sous la ligne "new" :

https://gist.github.com/271274

Modifier l'ordre des champs dans la vue list, et grouper les champs dans la vue new :

https://gist.github.com/271285

Changer le format des champs date :

https://gist.github.com/271313

Modifier le titre de chaque view :

https://gist.github.com/271314

Changer le lien vers la vue edit :

Par défaut, dans la view list, il y a un lien vers le formulaire de modification sur la colonne ID. Pour supprimer la colonne ID et mettre le lien sur une autre colonne :

https://gist.github.com/271316

Changer le mode d'affichage en view list

Par défaut, les données sont présentées sous forme de tableau. On peut cependant les afficher autrement, de cette façon :

https://gist.github.com/271319
