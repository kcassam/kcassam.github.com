php cli Mac OS X

November 2 2010, 11:02 AM  by Karim Cassam Chenaï
Q: J'ai MAMP, mais php ne fonctionne pas bien en ligne de commande
R: Vérifier que l'exécutable est pas appelé par le fichier appelé (exemple #!/usr/bin/php -q). Si OK, il faut appeler le bon exécutable, dans /Applications/MAMP/bin... Par exemple comme ceci : /Applications/MAMP/bin/php5.2/bin/php nomdufichier.php

Q: Comment savoir quel php j'utilise par défaut ?
R: taper 'which php', sans les cotes :$ which php
/usr/bin/php

(ça marche pour d'autres commandes)

Q: Comment éviter de taper le chemin complet /Applications/MAMP/bin/php5.x/bin/php à chaque fois ?
R: Faire un lien symbolique du php par défaut (obtenu via which php) vers le php que vous voulez utiliser. En ayant pris la précaution d'en faire une copie avant. Il y a d'autres solutions qui utilisent la variable path.

Q: Ça marche toujours pas : J'obtiens un message d'erreur 'Permission denied'
R: Vérifiez les droits sur le lien symbolique et sur le fichier ciblé.
